Overview

The DDoS Detection System is a real-time network traffic monitoring and analysis tool designed to detect Distributed Denial of Service (DDoS) attacks. It leverages an LSTM (Long Short-Term Memory) neural network to classify network traffic as normal or malicious, providing a web-based dashboard for monitoring and a RESTful API for integration with other systems.

Key Features

  • Real-time traffic analysis and attack detection
  • Machine learning-driven classification
  • Interactive monitoring dashboard
  • API endpoints for system interaction
  • Robust logging for diagnostics

System Components

Data Loader

Loads and preprocesses the CIC-DDoS2019 dataset from CSV files for training and prediction.

Model Training

Trains the LSTM model with preprocessed data, optimizing for attack detection accuracy.

Prediction Model

Analyzes incoming traffic data using the trained LSTM model to identify DDoS attacks.

Web Server

Hosts API endpoints and serves the monitoring dashboard using Flask.

Attack Simulation

Generates controlled traffic to test the system's detection capabilities.

Main Script

Coordinates system initialization and optional model training.

Installation and Setup

  1. Prerequisites:
    • Python 3.8+
    • Libraries: numpy, pandas, tensorflow, scikit-learn, flask, matplotlib, seaborn, requests, chart.js, three.js
    • Install via pip install -r requirements.txt
  2. Dataset:
    • Place CIC-DDoS2019 CSV files in the ./data directory.
  3. Running the System:
    • Train a new model: python main.py --train
    • Start the server: python main.py
    • Access at http://localhost:5000

Usage

Monitoring

Navigate to /monitor to view real-time traffic statistics and detection status via the dashboard.

Testing

Trigger a controlled test via the /test-attack endpoint or the dashboard's "Run Test" button.

Training Process

The LSTM model is trained to detect DDoS attacks with high accuracy. The process includes:

1. Data Preparation

  • Loads CIC-DDoS2019 dataset from ./data.
  • Encodes labels (normal: 0, attack: 1) using LabelEncoder.
  • Drops irrelevant columns (e.g., 'Timestamp').
  • Hashes IPs to integers, cleans NaN/infinite values, and scales features with StandardScaler.

2. Sequence Generation

  • Creates balanced sequences (length: 10) from features, capped at 10,000 samples per class.
  • Shuffles sequences to remove order bias.

3. Model Construction

  • Builds an LSTM model with two LSTM layers (32 and 16 units), Dense layers, dropout (50%), and L2 regularization.
  • Compiles with Adam optimizer (lr=0.0005) and binary cross-entropy loss.

4. Training Execution

  • Splits data: 60% train, 20% validation, 20% test.
  • Uses callbacks: EarlyStopping, ReduceLROnPlateau, ModelCheckpoint.
  • Trains for 10 epochs with class weights to balance classes.

5. Evaluation and Saving

  • Evaluates on test set, saves metrics (e.g., accuracy: 98.16%) and model as ddos_model.h5.
  • Generates confusion matrix and learning curves.

API Documentation

GET /status

Retrieve current system status and metrics.

Response: JSON with requests per second, status, and total requests.

POST /predict

Analyze traffic data for DDoS detection.

Request Body: JSON with traffic_data matching training feature structure.

GET /model-metrics

Fetch model performance metrics.

Response: JSON with accuracy, precision, recall, and F1-score.

Dashboard Guide

Key Metrics

  • Current RPS: Real-time requests per second
  • Attack Probability: Likelihood of current traffic being malicious
  • Total Requests: Cumulative request count

Visualizations

  • Traffic Analysis: Plots RPS and attack probability over time
  • Detection Confidence: Shows model confidence levels
  • Feature Importance: Highlights key detection features

Troubleshooting

No Model Found

Run python main.py --train to train a model if none is detected.

Dataset Errors

Verify CSV files in ./data include a label column.

Server Issues

Check server.log and train_model.log. Ensure port 5000 is free.
